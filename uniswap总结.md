理解什么是AMM，与订单簿的区别  
AMM:自动做市商。通过数据公式x*y=k（k固定）来定价，需要lp提供流动性的一种做市方式。  
与订单簿的区别：   
| 特征         | 订单簿                | AMM       |   
| ------    | ------------- | --------- |   
| 交易对象   | 用户 vs 用户      | 用户 vs 资金池 |   
| 定价方式   | 挂单价格          | 数学公式      |   
| 流动性      | 做市商提供         | LP提供      |  
| 是否需要挂单 | 必须            | 不需要       |  
| 是否有深度  | 有 order depth | 没有        |  
| 滑点来源   | 深度不足          | 曲线价格      |  

b：AMM的核心算法是什么  
x*y=k。  用数学函数维持池子状态。并通过池子比例自动决定价格    

c：什么是滑点，用户交易时怎么避免滑点过高（三明治攻击？怎么防止？）  
滑点是因为公式 x * y=k 产生的，当用户交易需要买入x时，池子里的x减少，但公式规定K固定值，  
故y，即x的价格就要上涨，上涨的这部分就是滑点，当交易量占池子中x总量偏高时，滑点就会偏高。  
避免滑点过高的几种方法：1.设置滑点上限。2.拆分交易，把大额交易拆分成多比小额交易。3.使用交易聚合器。  
三明治攻击：套利机器人通过扫描mempool中的交易，如发现大额交易，就发起交易，并提高gas费，保证交易在
大额交易前完成，因为大额交易会推高价格，故机器人会在大额交易完成后，立刻卖出之前买入的资产，从而达到
套利的目的。需要两个前提条件：1.mempool是公开透明的。2.高gas可以优先完成交易。
防止套利被夹：1. 设置较低的滑点 2.使用私有交易, 机器人扫不到。3.拆分交易，4.聚合器。  

d：LP的作用是什么？  
AMM 能运转的核心角色。如果没有 LP，AMM 的池子里就没有资产，也就 无法交易。  

e：什么是无偿损失？怎么计算？  
无偿损失即LP因为AMM的定价算法被动卖掉了价格上涨的资产，导致持有的资产市值相比一直持有少，这部份差价叫
无偿损失。  
具体计算：币种占比保持不变并持有市值-提供LP币种占比发生变化后的市值。  

V3相对V2有了比较大的更新，主要是在于集中流动性、灵活的手续费等，需要掌握以下几个问题     
 a：集中流动性的优势是什么，怎么实现的？  
 V3集中流动性的优势是LP的资金利用率更高，交易深度更高，用户的滑点更低，LP能获得更多的交易费。  
 V2的价格区间是0到无穷大，但真实情况是价格只会落在一个小区间，lp的资金使用率低。tick是一个价格刻度，
 tickupper 与ticklower决定了一个价格区间，也就是一个LP position。
 
 b:  tick跟价格有什么关系，为什么需要tick ？  
定价公式是price=1.0001^tick，LP可以设置每一个LP position，如   
upperTick=1000, lowerTick=800, 那通过公式计算对应流动性价格的范围。
LP可以将资金集中在这个区间来提供流动性，提高资金利用率。  

c：tickspacing跟费率有什么关系，为什么?    
费率决定tickapcing。高费率->大tickapcing。
费率是由资产价格波动性决定的。
价格波动性 ->LP的风险->手续费->费率-> tickspacing。
1）资产价格稳定性越高，因为价格波动小，LP提供的流动性只需要集中在一个小的范围内，即tickspacing小    
这样，交易深度高，LP的资金利用率就高，单位资金收益高，滑点低，手续续可以低  费率自然低。 所以为了精细做市，  
tickspacing可以很小。

2）资产价格波动越高，LP风险越高(例如无偿损失)，LP就要求更高的手续费，费率也就越高，tickspacing就越大：因为波动高的资产，  
价格区间往往跨度大，甚至一个swap可能跨不同的tick，导致LP需要的资金相对多，铺到不同的tick中，提供流动性，  
这样资金利用率就低，且管理难度也会增加(要跟随价格，不断rebalance)，且跨tick的gas费也会增加，故需要tickspacing更大，来均衡 swap中  
跨tick, 资金利用率，gas费几个问题。
事实上tickSpacing是用来解决以下三个核心问题：  
1.解决资金碎片化的问题    
	如果不控制tickSpacing，LP为了追求资金使用效率，提供非常小的tickSpacing，导致swap的过程中不断cross tick, Gas会变高。  
2.控制tick的数量   
	tick的取值范围是固定的（-887272 → 887272），tickSpacing不同的取值，会导致tick的数量相差很大，会导致swap中寻找tick的工作量差别很大。   
3.降低LP策略复杂度    
 LP策略： 低波动 ->小tickspacing，高波动 ->大tickspacing。  
 高波动配小tickspacing，LP需要频繁调整策略。  
 低波动配大tickspacing，LP资金使用效率低。
