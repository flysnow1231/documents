第一步：别升级，先原样安装

有 lock 就：

npm ci

没 lock 就：

npm install
第二步：看配置和测试栈

重点看：

hardhat.config.*

test/*.ts

package.json

判断它到底是：

viem 还是 ethers

mocha 还是 node:test

Hardhat 2 还是 3

第三步：只做“最小修复”

不要一上来就：

升 Hardhat

升 OZ

升 TypeScript

重装所有插件

否则很容易把一个本来能跑的老工程搞成全红。

第四步：先跑 compile，再跑 test

顺序：

npx hardhat compile
npx hardhat test

编译都过不了，先别管测试。
